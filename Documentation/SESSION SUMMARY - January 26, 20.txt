ðŸ”¬ TECHNICAL DECISIONS LOG
Why separate NetworkManager and LobbyManager GameObjects?

FishNet architecture: NetworkManager = framework component (no NetworkObject)

LobbyManager = game logic (requires NetworkObject for RPCs)

Violation causes "NetworkObject on NetworkManager" error

Why use LoadSceneMode.Single instead of Additive?

Menu has video background that remains visible if additive

Single mode unloads previous scene automatically

Cleaner memory management

Why ownership-based interaction instead of spawning player-specific UI?

Two static panels simpler than dynamic spawning

Server can update both panels simultaneously

Client only controls their assigned panel

Clear visual feedback (dimmed non-owned panel)

Why check NetworkManager.IsClientStarted instead of LobbyManager.IsSpawned?

Scene NetworkObjects don't reliably report IsSpawned

NetworkManager state is authoritative

Modern FishNet API (IsClientInitialized) more reliable
